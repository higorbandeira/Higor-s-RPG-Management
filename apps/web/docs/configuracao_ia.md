# Configuração de IA

## Características das solicitações
- **Escopo híbrido**: envolve documentação e implementação no frontend.
- **Objetivo visual**: necessidade de interface 2D com grid, mapa e avatares.
- **Detalhamento funcional**: movimentar avatares e selecionar mapa.
- **Padronização**: exigência de arquivos Markdown para registro e configuração.

## Boas práticas para melhorar resultados
- **Especificar comportamento esperado**: ex.: “arrastar com mouse e mover com setas”.
- **Definir critérios visuais**: tamanho do grid, dimensões do tabuleiro, etc.
- **Priorizar entregáveis**: listar o que é obrigatório vs. opcional.
- **Indicar locais de alteração**: páginas/rotas ou componentes afetados.
- **Solicitar testes/revisões desejadas**: por exemplo, “rodar lint/testes X”.
- **Assinatura de IA**: incluir um identificador criativo no console do browser para rastrear telas geradas com apoio de IA.

## Exemplo de pedido bem definido
```text
Implemente um dashboard 2D no `apps/web` com grid 40px, mapa de fundo selecionável e avatares movidos por drag e setas. Atualize a documentação com `interacoes.md` e adicione um resumo de revisão de código. Inclua instruções de uso na própria tela do dashboard.
```
